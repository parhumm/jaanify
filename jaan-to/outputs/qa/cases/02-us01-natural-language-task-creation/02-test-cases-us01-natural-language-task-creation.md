- AI parsing service (OpenAI GPT-4o-mini) accessible with test API key
- PostgreSQL database with task schema provisioned
- Test user account: freelancer@example.com (created during test data setup)
- Modern browser: Chrome 120+, Firefox 120+, or Safari 17+
- Network throttling tools available for timeout/disconnection tests

### Test Data Setup

- Create test user account "freelancer@example.com" with default permissions
- Ensure AI parsing service health endpoint returns 200
- Clear any existing tasks for the test user
- Configure character limit to 500 (as per US-01 open question suggestion)
- Set AI parsing debounce to 300ms
- Set AI parsing timeout to 5000ms

### Shared Preconditions

- Application is running and accessible
- AI parsing service is healthy (unless explicitly stated otherwise in the scenario)
- User is authenticated as "freelancer@example.com"
- User is on the task input screen
- No pre-existing tasks in the test user's account

### Execution Order

Tests are **independent** and can be executed in any order. However, for optimal results:

1. **Smoke suite first**: Run tests tagged @smoke to verify critical paths
2. **Regression suite**: Run all tests tagged @regression
3. **Priority-based**: Execute @priority-critical, then @priority-high, then @priority-medium/low

### Test Data Guidelines

All test cases use **concrete data values** (not placeholders):

- **Emails**: freelancer@example.com, test@example.com
- **Task inputs**: "Call Sarah about the Johnson proposal by Friday 2 PM", "Think about new marketing strategy", etc.
- **Dates**: Friday 2 PM, March 15, next Tuesday at 3 PM
- **Numbers**: 0, 1, 499, 500, 501 (character limits), 100, 200, 300, 4999, 5000, 5001 (timeout ms)
- **URLs**: Jaanify app test environment URL

These values are **safe for test environments** and should be created/cleaned up per test run.
